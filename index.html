CSS METHODOLOGY

For our project: We will be utilising components of both OOCSS and BEM.
The reason being:

• TO Reduce the bloating of classes within the HTML
• TO Have multiple selectors/styles that can be reused throughout our code on a variety of different elements.

OOCSS

Object Oriented CSS is an approach for writing CSS that encourages you to think about your style-sheets as a collection of "objects".

It allows for Reusable, Repeatable code snippets that can be used across the website on a variety of different elements.

The main purpose of OCSS is to create style-sheets that are easier to maintain and add to.

A main principle for using OOCSS is to limit the amount of ID's used to style single elements.

Example

< header class="wrapper">
< button class="wrapper">

employs the style:

.wrapper {
margin: 10px;
}

< header>

BEM

BEM, or “Block-Element-Modifier”, is a naming convention for HTML and CSS classes.
It is a way of naming your CSS classes to give them more transparency and meaning to other developers.

They are far more strict and informative at a glance.

Example

/* Block component */
.btn {}

/* Element that depends upon the block */
.btn__price {}

/* Modifier that changes the style of the block */
.btn--pink {}
.btn--big {}

How this appears in the HTML

"< button class="btn btn--big btn--pink"> < /button>"

COLOUR CLASSES

For our project: We will be incorporating a dark and light theme.

• We will be utitlising SASS within our application as it provides a simpler, more elegant syntax for CSS.

• We are able to create $variables for our colours, and @mixins to group declarations within our code.
